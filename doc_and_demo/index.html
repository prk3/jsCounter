jCounter

Easy to use and powerful plugin for creating counters (of any kind)

Counter initialization

To use this plugin you just need to include jcounter.min.js or counter.js in your webpage.

Linking jCounter script
<script src="../jcounter.min.js"><script>

Counters can be initialized in two ways. The first one is the global function "jCounter". It takes as a first argument DOM elements, in which numbers will be put and as a second one javascript object with counter parameters.

Example usage of jCounter function
jCounter(document.getElementsByClassName('counter'), {
	start: 200,
	end: 400,
	animation: 'ease-in'
});

The second method of counter initialization relies on jQuery and will be available if jQuery was included before jCounter.

Same result using jQuery syntax
$('.counter').jCounter({
	start: 200,
	end: 400,
	animation: 'ease-in'
});

Counter customization

Counters can be customized by passing object with counter parameters to initialization function. All of the parameters can be also set using data-attributes in html element. Data-attributes have higher priority than parameters in jCounter settings object. Below examples will have the same result.

1
<div class="counter"></div>
$('.counter').jCounter({
	start: 200,
	end: 400,
	animation: 'ease-in'
});
2
<div class="counter" data-start="200" data-end="400" data-animation="ease-in"></div>
$('.counter').jCounter();

Write data-attribute names in lowercase. For example:
updatesPerSec: 10 -> data-updatespersec="10"

You can not pass functions in data-attributes.

Counter options

This is a list of all counter parameters you can use.

  • start - starting number

    Default:
    0

  • end - last number

    Default:
    100
    start and end example

    Simple counter

    var config = {
	start: 0,
	end: 240
};
    Start

    From a negative to a positive

    var config = {
	start: -150,
	end: 150
};
    Start

  • duration - how many miliseconds counting will take

    Default:
    2500

    Real animation duration may differ when duration is not a multiple of updatesPerSec. Difference will not be grater than 1/updatesPerSec seconds.

  • updatesPerSec - describes how frequently counter is being updated.

    Default:
    18

    Values higher than 25 may cause performance drop.

    duration and updatesPerSec example

    Short duration and low update frquency

    var config = {
	duration: 1500,
	updatesPerSec: 10
};
    Start

    Long duration and high frequency

    var config = {
	duration: 6000,
	updatesPerSec: 25
};
    Start

  • animation - function which defines the way in which starting number grows (or shrinks) to final number.

    Default:
    'linear'

    Built in animations:

    • 'linear'
    • 'ease-in'
    • 'ease-out'
    • 'ease-in-out'

    You can define own function as animation parameter. It will be called with two arguments: 1st - progress of animation as a number between 0 and 1 (including 0 and 1), 2nd - animationPower variable (see animationPower option).

    Custom animation functions should fulfill two rules:
    f(0) = 0
    f(1) = 1
    Otherwise counter will not count from start to end parameters.

    'linear' animation is perfect for creating timers or when cpu-light animation is needed.

    See more about animation functions:
    http://easings.net
    https://gist.github.com/gre/1650294#file-easing-js-L7

  • animationPower - number which is passed to animation function as a second argument. 'ease-in', 'ease-out', 'ease-in-out' animations with animationPower = 1, 2, 3 will calculate numbers using functions respectively: linear, quadratic, cubic, etc. animationPower can be also a floating point number. The higher animationPower is, the "smoother" counting will be.

    Default:
    2

    Use 'linear' animation instead of ease animations with animationPower = 1. I gives same result and consumes less computational power.

    animations example

    Linear animation

    var config = {
	duration: 4000,

	animation: 'linear'
};
    Start

    Gentle ease-in-out 

    var config = {
	duration: 4000,

	animation: 'ease-in-out',
	animationPower: 2
};
    Start

    Slippery ease-in-out

    var config = {
	duration: 4000,

	animation: 'ease-in-out',
	animationPower: 4
};
    Start

    Smooth ease-out

    var config = {
	duration: 4000,

	animation: 'ease-out',
	animationPower: 3
};
    Start

    Own "overflow" function

    var config = {
	duration: 4000,

	animation: function(x, power) {
		return -1.6 * Math.pow(x, power) + 2.6 * x;
	},
	animationPower: 2.5
};
    Start

  • digitsOnRight - number of decimal places shown. If digitsOnRight is smaller or equal to 0 decimal places are not displayed.

    Default:
    0

  • digitsOnLeft - number of places reserved on left side of decimal separator. It's ignored when number occupies more places than digitsOnLeft states.

    Default:
    0

  • leftFiller - character or string which will be put in reserved places not filled by number (see digitsOnRight parameter).

    Default:
    '0'
    digitsOnLeft, Right and leftFiller example

    4 reserved places, 2 decimal

    var config = {
	end: 1100
	duration: 3000,

	digitsOnLeft: 4,
	digitsOnRight: 2,
	leftFiller: '0'
};
    Start

    Underscore filler

    var config = {
	start: 1,
	end: 12345,
	duration: 6000,

	animation: 'ease-in',

	digitsOnLeft: 5,
	digitsOnRight: 0,
	leftFiller: '_'
};
    Start

  • decimalSeparator - character or string which will be used as decimal separator.

    Default:
    '.'

  • groupSeparator - character or string which will be put every nth digit of whole number part starting from right, where n is groupWidth (see groupWidth option).

    Default:
    '&nbsp;'

  • groupWidth - number of digits in groups separated by the groupSeparator (see groupSeparator option).

    Default:
    3
    • separators example

    US and UK notation

    var config = {
	end: 1404260,
	duration: 8000

	animation: 'ease-in',

	digitsOnRight: 2,
	digitsOnLeft: 7,
	leftFiller: '0',

	decimalSeparator: '.',
	groupSeparator: ',',
	groupWidth: 3
};
    Start

    European notation

    var config = {
	end: 1404260,
	duration: 8000

	animation: 'ease-in',

	digitsOnRight: 2,
	digitsOnLeft: 7,
	leftFiller: '0',

	decimalSeparator: ',',
	groupSeparator: '&nbsp;',
	groupWidth: 3
};
    Start

    Chinese grouping

    var config = {
	end: 1404260,
	duration: 8000

	animation: 'ease-in',

	digitsOnRight: 2,
	digitsOnLeft: 7,
	leftFiller: '0',

	decimalSeparator: '.',
	groupSeparator: ',',
	groupWidth: 4
};
    Start

  • separateFillers - treat fillers the same way as digits - put decimal separator every third filler

    Default:
    true

  • showPlus - show plus sign '+' in front of positive numbers

    Default:
    false
    separateFillers and showPlus example

    Do not separate fillers

    var config = {
	end: 1404260,
	duration: 8000,

	animation: 'ease-in',

	digitsOnLeft: 7,
	groupSeparator: '`',
	leftFiller: '_',

	separateFillers: false
};
    Start

    From -1100 to 1100 with plus sign

    var config = {
	start: -1100,
	end: 1100,
	duration: 8000,

	showPlus: true
};
    Start

  • onStart - function, which will be executed right before start of a counter.

    Default:
    undefined

  • onEnd - function, which will be executed right after the end of counting.

    Default:
    undefined
    onStart and onEnd example

    Change some css

    var config = {
	onStart: function(element) {
		//classic javascript styling
		element.style.color = '#ffbe00';
		element.style.fontWeight = 'bold';
	},

	onEnd: function(element) {
		//jQuery styling
		$(element).css({
			'text-shadow': '2px 2px 5px #ffbe00',
			'text-decoration': 'underline'
		});
	}
};
    Start

  • initializeOnly - do not start counting, just fill element with starting value.

    Default:
    false

    All examples on this page except for one below were initialized on pageload with "initializeOnly: true", so that you can see how do they look like before start of counting.

    initializeOnly example

    Click start to initialize counter

    var config = {
	initializeOnly: true
};
    Start

Counter HTML elements

Each element of a counter (digit, separator) is enclosed in separate HTML element.

Example counter element with jCounter 
<div class="counter">
	<span class="jc-pp3">9</span>
	<span class="jc-gs">`</span>
	<span class="jc-pp2">9</span>
	<span class="jc-pp1">9</span>
	<span class="jc-pp0">9</span>
	<span class="jc-ds">.</span>
	<span class="jc-pm1">9</span>
	<span class="jc-pm2">9</span>
</div>

Below table explains which classes are given to all counter elements

Counter element Class name
digitpositive power jc-pppower
digitnegative power jc-pmpower
sign + jc-sp
sign - jc-sm
decimal separator jc-ds
group separator jc-gs
left filler jc-lf

First digit on the left of decimal separator has class name "jc-pp0". Next one on the left will have class "jc-pp1", next one "jc-pp2" and so on. First digit on the right of decimal separator has class name "jc-pm1", next on the right "jc-pm2", next "jc-pm3" and so on. If we sum (digit * 10digit's power) for all of the digits we would get a value of the displayed number.

Because every counter's element has own class name you can style a counter in any way you can imagine.
